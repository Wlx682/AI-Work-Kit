# 示例：创建自动化任务 — PRD 问题模式

> **不是 PRD 原文**，是从典型 PRD 结构里拆出的**问题类型示例**，供分析时对照。完整 PRD 在任务对话里贴链接。

## PRD 结构摘要（分析输入）

- 模块：定时任务管理 · 创建自动化任务表单
- 频率：周期 / 间隔 / 单次
- 入口：`+`（有任务时）、空态「立即创建」、案例卡片预填、详情「编辑」复用创建页
- 表单：龙虾 → 名称 → 具体要求(必填) → 模型 → 频率 → 时间窗口 → 描述
- 创建按钮：具体要求非空 + 频率配置完整后激活

## 已识别的逻辑/交互问题（示例）

### L1 · 入口条件不对称 【P1】

| | |
|--|--|
| **PRD** | 「+」**有**自动化任务时显示，**没有**时不显示；空态另有「立即创建」 |
| **冲突** | 两套入口依赖相反条件，需确认是否故意（空态引导 vs 列表内快捷） |
| **问产品** | 有任务后是否隐藏空态？仅保留「+」？Tab 切换后「+」是否仍可见？ |

### L2 · 创建页 vs 编辑页 【P0】

| | |
|--|--|
| **PRD** | 详情「编辑」→ 复用创建页，预填数据 |
| **冲突** | 导航标题写「创建自动化任务」？主按钮写「创建」还是「保存」？校验是否与创建一致（如频率不可改？） |
| **问产品** | 编辑态标题/按钮文案；哪些字段只读；保存是否调 update API |

### L3 · 任务名称非必填 【P1】

| | |
|--|--|
| **PRD** | 任务名称非必填 |
| **冲突** | 列表卡片、搜索、批量操作展示什么？与「具体要求」第一行关系？ |
| **问产品** | 列表主标题取值优先级；空名称是否用具体要求截断 |

### L4 · 创建按钮激活条件 【P0】

| | |
|--|--|
| **PRD** | 具体要求非空 + **频率配置完整** |
| **冲突** | 「完整」在周期/间隔/单次下定义不同；时间窗口是否参与激活？ |
| **问产品** | 每种频率模式的必填子字段表；时间窗口为空是否允许 |

### L5 · 案例卡片预填 【P1】

| | |
|--|--|
| **PRD** | 点击案例卡片自动填入参数 |
| **冲突** | 预填后是否覆盖用户已输入？是否算「已修改」？返回是否清表？ |
| **问产品** | 预填行为；是否进入创建页还是当前页弹层 |

### L6 · 「+」仅自动化 Tab 【P1】

| | |
|--|--|
| **PRD** | 「+」只有切换到自动化 Tab 才显示 |
| **冲突** | 全部 Tab 里也有定时任务卡片时，用户能否从全部 Tab 创建？ |
| **问产品** | 与现有 `NMTaskRecordListVM` 多 Tab 行为对齐策略 |

### L7 · 与代码库现状 【待查】

ClawAI 已有自动化 Tab、`cron.list`、task-history 等，分析时应搜索：

- 列表刷新、创建成功后通知（如 `nmTimedTaskListItemChanged`）
- 模型列表是否与「输入框模型列表一致」同一数据源

## 整体需求遗漏（示例）⭐

> PRD 只写了「创建表单」，以下环节**全文未提及或明显不全**时，应列入遗漏表 push 产品。

### 用户旅程闭环

| 环节 | 典型遗漏 |
|------|----------|
| 创建成功 | 跳列表还是详情？是否 Toast？ |
| 创建失败 | 错误文案、是否保留表单内容 |
| 未保存返回 | 是否二次确认、是否草稿 |
| **任务详情** | 仅提「详情页编辑」，详情展示字段、启停按钮未在本 PRD |
| **删除任务** | 常整段缺失 |
| **启用/暂停** | 定时任务几乎必做，PRD 常不写 |
| 列表刷新 | 创建/编辑后自动化 Tab、全部 Tab 如何更新 |

### 频率模式细则 【P0 遗漏高发】

PRD 写「周期、间隔、单次」但常未写：

- 每种模式的子字段表（周几、几点、间隔单位、单次日期时间）
- 模式切换时已填内容是否清空
- 非法组合（结束时间早于开始时间）如何校验

### 执行时间窗口 【P1】

- 与频率的关系：是过滤执行还是仅展示？
- 跨天、时区、默认全天是否允许

### 异常态 【P1】

- 龙虾列表/模型列表拉取失败
- 无可用龙虾、模型被下线
- 保存中 loading、防连点
- 网络超时重试

### 接口与数据 【P0】

- create / update / list 接口文档
- 错误码与用户提示映射
- 任务数量上限、名称/具体要求长度上限

### 关联与验收 【P1–P2】

- 案例卡片数据来源、运营是否可配置
- 埋点：创建成功、失败、各入口点击
- 验收用例最低集合
