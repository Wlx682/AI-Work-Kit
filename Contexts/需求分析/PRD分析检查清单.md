# PRD 分析检查清单

> 需求质量参差时，**先分析、再开发**。目标三件事：**逻辑矛盾**、**交互冲突**、**整体遗漏** → 整理成可 **push 产品** 的确认清单。

## 何时做

- 收到飞书 PRD / 口头需求 → **先** `/requirement-analyst`
- P0 未闭环（含**关键遗漏**）→ **不** 进 `/feature-dev-assistant` 写代码

## 分析七步

```
1. 摘要 — 用自己的话复述要做什么（防误读）
2. 入口矩阵 — 谁、在哪、什么状态下、进哪页
3. 字段规则 — 必填、默认值、按钮激活条件
4. 逻辑扫描 — 矛盾、未定义分支（写了但说不通）
5. 交互扫描 — 空态/有数据、创建/编辑、Tab 差异
6. 遗漏扫描 — PRD 根本没写、但完整功能应有的部分 ⭐
7. 对照代码库 — 与现有实现是否冲突、是否已有可复用行为
```

---

## 一、逻辑问题（写了，但有问题）

| 检查项 | 问什么 |
|--------|--------|
| 条件互斥 | A 说显示，B 说隐藏，能否同时成立？ |
| 必填 vs 展示 | 非必填字段为空时，列表/详情怎么展示？ |
| 激活条件 | 保存按钮条件是否覆盖所有频率模式？ |
| 复用页面 | 创建/编辑共用页时，标题、按钮、校验是否一致？ |
| 默认值异常 | 默认模型/龙虾不存在或下线时怎么办？ |
| 编辑副作用 | 编辑后是否影响正在执行的任务？ |
| 案例预填 | 预填与用户已输入冲突时以谁为准？ |

---

## 二、交互冲突（写了，但互相打架）

| 检查项 | 问什么 |
|--------|--------|
| 入口不对称 | 空态有入口、有数据时反而没有（或相反）？ |
| Tab 隔离 | 控件「仅某 Tab 显示」，其它 Tab 如何完成同一目标？ |
| 返回栈 | 保存后回哪？未保存返回是否提示？ |
| 状态刷新 | 创建/编辑成功后列表、空态、角标是否同步？ |
| 与全局一致 | 模型列表、龙虾列表是否与 App 其它入口同源？ |

---

## 三、整体遗漏（没写，但应该有）⭐

> **与「逻辑问题」区别**：逻辑问题是 PRD **写了两处互相矛盾**；遗漏是 PRD **整段缺失**，开发只能猜。

### 3.1 用户旅程是否闭环

对照本功能画一条链路，逐项打勾 PRD 是否写明：

| 环节 | 是否写明 | 常见遗漏 |
|------|----------|----------|
| 进入 / 退出 | | 从哪进、返回哪、是否保留草稿 |
| 创建 | | 仅写了表单，没写提交后跳哪 |
| 成功反馈 | | Toast / 回列表 / 进详情？ |
| 列表展示 | | 字段、排序、分页、空态 |
| 详情 | | 是否有独立详情 PRD？ |
| 编辑 | | 与创建差异、不可改字段 |
| 删除 / 停用 / 启用 | | 定时任务类几乎必漏 |
| 搜索 / 筛选 | | 列表有无搜索框 PRD 未提 |
| 失败重试 | | 保存失败、加载失败 |

### 3.2 表单类：频率 / 模式 / 子配置

| 检查项 | 问什么 |
|--------|--------|
| 每种模式细则 | 周期/间隔/单次各自有哪些子字段、默认值、校验？ |
| 模式切换 | 从 A 模式切到 B，已填数据是否清空？ |
| 时间与时区 | 执行时间、时间窗口、夏令时、跨天 |
| 边界值 | 间隔最小值、单次过期、名称最大长度 |

### 3.3 状态与异常（PRD 常整段缺失）

| 状态 | PRD 是否描述 UI + 行为 |
|------|------------------------|
| 加载中 | |
| 空数据 | |
| 请求失败 / 超时 | |
| 无权限 / 未登录 | |
| 服务不可用 | |
| 字段校验失败 | |
| 保存中（防连点） | |
| 部分字段加载失败（如下拉列表） | |

### 3.4 数据、接口、持久化

| 检查项 | 问什么 |
|--------|--------|
| API 清单 | 创建/更新/列表/详情/删除/启停各用什么接口？ |
| 错误码 | 失败时用户看到什么？可否重试？ |
| 幂等 | 连点创建是否重复？ |
| 离线 / 弱网 | 是否支持？不支持是否说明？ |
| 与后端约束 | 任务数上限、频率限制、敏感词 |

### 3.5 与关联模块（PRD 常默认「你们懂」）

| 检查项 | 问什么 |
|--------|--------|
| 上游依赖 | 龙虾列表、模型列表从哪来？为空怎么办？ |
| 下游影响 | 创建后哪些页面要刷新？通知机制？ |
| 同类功能对齐 | 与 App 内其它表单/列表交互是否一致？ |
| 多端 | 仅 App？PC 同 PRD 还是另文档？ |

### 3.6 非功能与验收（常被整段遗漏）

| 检查项 | 问什么 |
|--------|--------|
| 成功指标 | 怎么算做完？ |
| 埋点 | 关键行为是否要求上报？ |
| 无障碍 | 读屏、最小点击区（iOS 44pt） |
| 性能 | 列表上限、首屏时间 |
| 安全 / 合规 | 用户输入内容是否审核、隐私 |
| 验收用例 | 有无最低测试场景列表？ |

### 3.7 遗漏输出格式

| # | 遗漏维度 | PRD 覆盖情况 | 缺失内容 | 不补的风险 | 严重度 |
|---|----------|--------------|----------|------------|--------|
| M1 | 用户旅程 | 未写 | 【】 | 开发猜导航/刷新 | P0/P1/P2 |

## 四、输出给产品的格式

每条问题（含遗漏）包含：

1. **类型**：逻辑矛盾 / 交互冲突 / **整体遗漏**
2. **现象**（用户或开发会遇到什么）
3. **PRD 现状**（摘录或「全文未提及」）
4. **建议补全方向**（2–3 个可选方案）
5. **严重度** P0 阻塞 / P1 建议 / P2 知晓

**Push 产品时分三块列出**：逻辑问题 · 交互冲突 · **需求遗漏**

## 沉淀原则

| 沉淀 | 不沉淀 |
|------|--------|
| 检查清单、遗漏维度、共性踩坑 | 飞书 PRD 全文 |
| 某类需求的**问题模式**（脱敏） | PRD 链接 |
| 分析结论摘要 | 进行中的 `Plans/需求分析/` 做完可删 |
