# 公共设计规范：360安全龙虾（ClawAI）

> 目标：**1:1 还原设计图**。数值以 Figma 节点为准；本文件只提供**未标注时的兜底值**和工程约定。

## 核心原则

| 优先级 | 原则 |
|--------|------|
| 1 | **设计稿节点属性优先** — 尺寸、圆角、描边、阴影、颜色、间距，从 Figma **逐个节点读取** |
| 2 | **禁止靠截图估** — 不根据截图目测像素；用 Figma MCP / Dev Mode / Inspect 读具体 layer |
| 3 | **@2x 画板换算** — 设计标注为 px（@2x）→ 代码 `pt` = **px ÷ 2** |
| 4 | **规范值仅兜底** — 设计未标注时，才用下文「默认值表」 |
| 5 | **状态稿必读** — 默认 / 按压 / 禁用等，有独立节点或 Variant 的必须逐态读取，不臆造 |

## 尺寸换算（@2x 画板）

```
Figma 节点 width/height/padding/gap/fontSize (px) ÷ 2 = iOS 代码 (pt)
```

| Figma (px) | iOS (pt) |
|-----------|----------|
| 16 | 8 |
| 24 | 12 |
| 32 | 16 |
| 34 | 17 |
| 42 | 21 |
| 72 | 36 |

**禁止**：把 Figma px 直接当 pt；**禁止**：不读节点、按经验凑数。

**读取顺序**：选中 Frame → 子节点自上而下（或按 Auto Layout 层级）→ 记录每层 `x/y/w/h`、padding、gap、constraints。

## 节点读取清单（每个可见元素）

开发前 / 写代码时，对每个 layer 确认：

| 属性 | 从 Figma 读什么 | 代码 |
|------|-----------------|------|
| 位置与尺寸 | frame、constraints、Auto Layout | SnapKit 约束 |
| 圆角 | `cornerRadius`（独立圆角看四个角） | `layer.cornerRadius` |
| 描边 | `strokeWeight`、`stroke` 颜色/透明度 | `borderWidth` / `borderColor` |
| 填充 | fill 颜色、渐变、透明度 | `backgroundColor` |
| 阴影 | offset、blur、spread、颜色 | `shadowOffset` 等（÷2） |
| 字体 | fontFamily、fontSize、lineHeight、letterSpacing | `UIFont` + 行高 |
| 间距 | padding、itemSpacing | 约束 constant |
| 状态 | Variant 名 / 独立 Frame（如 pressed） | 分态实现 |

有 **Component + Variant** 的，按 Variant 各读一遍，不要只实现 Default。

## 默认值表（仅设计未标注时使用）

> 设计稿有标注 → **以设计为准**。下表是「设计师没标、节点上也没有」时的兜底。

### 圆角

| 场景 | 兜底 (Figma px @2x) | iOS pt |
|------|---------------------|--------|
| 小按钮 / Tag | 8 | 4 |
| 卡片 / 弹层 | 16 | 8 |
| 全圆角胶囊 | 高度的一半 | 高度的一半 |

### 描边 (border)

| 场景 | 兜底 (Figma px) | iOS pt |
|------|-----------------|--------|
| 分割线 / 细边框 | 1 | 0.5 |
| 强调边框 | 2 | 1 |

### 阴影

| 场景 | 兜底（Figma px，需 ÷2 进代码） |
|------|-------------------------------|
| 轻阴影 | y: 4, blur: 8, color #000 @ 8% |
| 卡片浮起 | y: 8, blur: 16, color #000 @ 12% |

设计在 Effects 里写了具体值 → **用设计的，不用上表**。

### 按压态 / 交互态

| 规则 |
|------|
| 设计有 `pressed` / `active` Variant → **读该 Variant 的 fill、opacity、scale** |
| 设计无按压稿 → 与设计师确认；临时可用 opacity 0.6，**须在 plan 标「待设计确认」** |
| 禁止无稿自造按压色 |

### 颜色（仅无 fill / 无 style 时）

| 用途 | 兜底 |
|------|------|
| 背景 | `#FFFFFF` |
| 卡片底 | `#F5F5F5` |
| 主文案 | `#000000` @ 90% |
| 次要文案 | `#000000` @ 50% |
| 弱底/图标底 | `#000000` @ 4% |

节点上有 fill / Text Style → **用节点的**。

### 字体（仅无 Text Style 时）

- PingFang SC → `UIFont.systemFont`
- 常见：12pt / 14pt 正文档

## 开发约定

- 布局常量放 `private enum Layout`，**数值来自节点读取结果**，命名带语义（如 `cardCornerRadius`）
- 最小点击区域 **44×44 pt**（热区可大于视觉，视觉仍以设计为准）
- 画板命名：`MMDD-功能描述`
- 新任务 plan → `Plans/功能开发/`（做完可删；纯 UI 设含业务逻辑=否）
- 走查：**真机截图 vs Figma 同 Frame 叠图**，差异 > 1pt 要说明原因

---

## 与「Token 优先」的关系

1. **先 1:1 读出设计数值** → 写进 plan 的「节点度量表」
2. **再对照**代码里是否已有同名 Token / `NMColorHelper` — 能映射则映射，不能则按读出的值实现
3. 禁止跳过读节点、直接套规范表里的固定数字
