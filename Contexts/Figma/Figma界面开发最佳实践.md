# Figma 界面开发最佳实践

> 本文件 + [[项目设计规范]] + Templates 即全部常驻材料。不维护功能清单（已开发完的功能不沉淀）；链接与 Frame 细节任务时提供。

## 一、开发前必做（15 分钟，省 2 小时返工）

### 1. 设计稿信息清单

开工前 plan 里必须写齐：

| 项 | 示例 |
|----|------|
| Figma 链接 | **任务时由你在对话粘贴**（含 node-id）；只记入当次 plan，Contexts 不存 |
| Frame 名称 | `Settings / iOS / Default` |
| 平台 | iOS / Android / Web / 多端 |
| 设计版本 | v2.3（设计师确认日期） |
| 是否用 Dev Mode | 是 / 否 |

### 2. 组件复用检查（先查再写）

```
设计稿里的元素 → 对照代码库已有组件 → 标出「复用 / 扩展 / 新建」
```

| 设计元素 | 代码组件 | 动作 |
|----------|----------|------|
| Primary Button | `PrimaryButton` | 复用 |
| 列表 Cell 带副标题 | `SubtitleCell` | 复用 |
| 新样式分段控制器 | 无 | 新建 → 需设计确认是否入 Design System |

**原则**：能复用不新建；新建组件必须说明入不入组件库。

### 3. 逐节点读取（1:1 还原，禁止截图估）

**实现依据是 Figma 节点属性，不是截图目测。**

推荐流程：

```
1. 打开 Frame（带 node-id 的链接）
2. 用 Figma MCP / Dev Mode 逐个 layer 读：尺寸、圆角、描边、阴影、fill、字体、间距
3. @2x：所有 px ÷ 2 → pt 写入 plan「节点度量表」
4. 有 Variant（默认/按压/禁用）→ 每个 Variant 单独读一遍
5. 再对照代码库：能复用组件 / Token 则映射，不能则按读出值实现
```

| 属性 | 设计有标注 | 设计无标注 |
|------|------------|------------|
| 圆角、描边、阴影 | **用节点上的值** | 用 [[项目设计规范#默认值表（仅设计未标注时使用）]] |
| 按压态 | **读 pressed Variant** | plan 标「待设计确认」，勿自造 |
| 颜色 | 读 fill / Text Style | 兜底色表 |

plan 里维护 **节点度量表**（见模板），不要只贴一张截图。

### 4. 状态与边界（设计稿常缺，开发前主动确认）

每个页面/组件确认是否需实现：

- [ ] 默认态
- [ ] 加载中（骨架屏 / Loading）
- [ ] 空数据（Empty）
- [ ] 错误态（Error + 重试）
- [ ] 禁用态
- [ ] 深色模式（若项目支持）
- [ ] 超长文案截断规则
- [ ] 无障碍：对比度、最小点击区域（iOS 44pt）

## 二、推荐实施顺序（切片开发）

不要一次实现整页。按 plan 勾选：

```
1. 布局骨架（容器、滚动、安全区）
2. 静态 UI（无交互、无数据）
3. 数据绑定（ViewModel / 状态）
4. 交互（点击、跳转、动画）
5. 异常态 + 边界
6. 设计走查对齐
```

每完成一步在 plan 打勾，方便跨会话续做。

## 三、Figma → 代码对照规则

### Auto Layout → 布局

| Figma | iOS (SwiftUI) | Web (React) |
|-------|---------------|-------------|
| Vertical + gap | `VStack(spacing:)` | `flex-direction: column; gap:` |
| Horizontal + gap | `HStack(spacing:)` | `flex-direction: row; gap:` |
| Fill container | `frame(maxWidth: .infinity)` | `flex: 1` |
| Hug contents | 默认 / `fixedSize()` | `width: fit-content` |
| Padding | `.padding()` | `padding:` |

### 尺寸

- 所有可见尺寸 **来自节点**（@2x px ÷ 2 = pt）
- 宽度「Fill」→ 自适应，约束关系仍按 Auto Layout 读
- 高度「Hug」→ 由内容撑开，min/max 以设计为准
- 禁止不读节点、用规范表里的示例数字代替真实标注

### 切图导出

#### 图层命名：`**` = 需导出切图

本团队 Figma 约定：**图层 / Frame 标题被 `**` 包裹的，表示需要导出为工程资源**（图标、插画、背景图等），不是纯布局占位。

| 图层标题示例 | 含义 | 开发动作 |
|--------------|------|----------|
| `**icon_membershipexpirarion_pop_bg**` | 可导出位图/矢量 | 导出 @2x/@3x 或 PDF，写入 `Assets`，代码 `UIImage(named:)` |
| `标题文案`（无 `**`） | 布局 / 文本，用代码实现 | 读字号、色值、间距，**不导出** |

- 读稿时先筛带 `**` 的节点，列入 plan「切图清单」；其余用 UIKit 还原
- 导出文件名以去掉 `**` 后的图层名为准，或与设计确认 Assets 命名；导出前查项目是否已有同名资源

| 资源类型 | 格式 | 命名 |
|----------|------|------|
| 图标 | SVG / PDF（iOS 矢量） | `ic_功能_状态` |
| 位图 | @2x @3x PNG | `img_模块_描述` |
|  Lottie | .json | `lottie_模块_描述` |

## 四、与设计师协作

### 开发中

- 标注「实现疑问」直接评论在 Figma 对应节点
- 差异 > 2px 或色值不一致：先确认是设计更新还是实现 bug

### 开发后：设计走查

用 [[Templates/Figma设计走查模板]]，逐项勾选：

- 间距、字号、颜色与 token 一致
- 各状态已实现
- 真机截图 vs Figma 并排对比

走查结论存 `Contexts/Figma/走查记录/`。

## 五、AI 协作提示（给 Cursor）

开发时 AI 应：

1. **先读** plan + [[项目设计规范]] + 本文件
2. **通过 Figma MCP 读节点**（有链接时；配置见 [[Figma-MCP配置]]），输出 **节点度量表** + 组件映射表 + **`**` 切图清单**，**禁止仅凭截图猜像素**
3. 数值优先级：**Figma 节点 > 设计 Variant > 规范兜底表**；禁止跳过 1、2 直接用兜底
4. @2x 画板一律 **px ÷ 2** 再写约束
5. **按切片顺序**实现；走查时真机 vs Figma 对比，差异 > 1pt 要说明

## 六、常见踩坑（#复盘/踩坑）

| 踩坑 | 教训 |
|------|------|
| 没确认状态稿，上线只有默认态 | 开工前用 checklist 问设计师 |
| 新建组件与现有重复 | 必须先做组件 inventory |
| 看截图估像素，圆角阴影全错 | 必须逐节点读取；无标注才用规范兜底 |
| 没读按压 Variant，点击态不对 | 有 Variant 必读 pressed/active |
| 整页一次写完，走查改一大片 | 切片开发 + 每片走查 |
| Figma 链接没带 node-id | 链接必须精确到 Frame |
