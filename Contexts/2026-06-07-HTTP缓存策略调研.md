# 调研：HTTP 缓存策略（库存服务）

**日期**：2026-06-07
**背景**：库存服务 6/8 上线缓存优化后出现偶发超时（见 [[2026-06-10-API超时排查]]）

## 结论摘要

1. **推荐**：本地缓存 + 短 TTL（30s）+ 异步刷新，避免请求路径上的同步锁
2. **不推荐**：在请求热路径上用 `sync.Mutex` 做 cache stampede 防护
3. **监控**：必须加 cache hit rate、lock wait time 指标

## 关键数据

- 缓存命中可将 inventory 查询从 200ms 降到 5ms
- 锁竞争场景下 P99 可从 800ms 飙到 5s+

## 决策

- 若确认锁竞争是根因，回滚为「无锁 + 允许短暂脏读」方案
