# 概念：提示词工程（Prompt Engineering）

> **学习顺序：第 1–2 课核心。**

## 一句话

**结构化地告诉模型：你是谁、要做什么、输入是什么、输出长什么样、不能做什么。**

## 好 prompt 的结构（CRISPE 变体）

| 块 | 内容 | Kit 中的对应 |
|----|------|--------------|
| Context | 背景、约束 | plan 里的「已知信息」 |
| Role | 角色 | `.cursorrules` 开头 |
| Instruction | 具体任务 | `/template-generator 任务类型=...` |
| Specification | 输出格式 | 模板里的「AI 输出要求」 |
| Examples | 少样本示例 | Case 参考 [[分享包-快速开始]] |
| Evaluation | 如何算做对 | 验收标准、走查清单 |

## 技巧速查

| 技巧 | 何时用 |
|------|--------|
| 分步指令 | 复杂排查、方案设计 |
| 输出格式约束 | 要表格、要 JSON、要 plan 结构 |
| Few-shot | 给 1–2 个「输入→理想输出」样例 |
| Chain-of-Thought | 推理题：「先列假设再结论」 |
| 明确非目标 | 防止过度设计（方案模板 §1） |

## 和 Skill 的关系

- **单次 prompt**：聊天里现写
- **Skill**：把验证过的 prompt **固化**成 `Skills/xxx.md` 或 `SKILL.md`

你的 `resume_assistant` 就是「续做」场景的固化 prompt。

## 练习（建议自己做）

1. 用 [[Templates/学习笔记模板]] 学一节外课  
2. 把笔记整理 prompt 写成 Skill 草稿存 `Contexts/LLM学习/范例/`  
3. `/learn-assistant 考我，范围=提示词结构`
