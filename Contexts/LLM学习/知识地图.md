# LLM 应用知识地图

> 从「聊天」到「能干活」的演进路径。箭头表示**建议学习顺序**。

```
┌─────────────┐
│  大语言模型   │  token、训练、推理、上下文窗口、温度
└──────┬──────┘
       ▼
┌─────────────┐
│  提示词工程   │  role、结构、few-shot、CoT、输出格式
└──────┬──────┘
       ▼
┌─────────────┐     ┌──────────────┐
│ 上下文管理   │────▶│  RAG / 检索   │  知识库、向量、BM25、你的 enquire-mcp
└──────┬──────┘     └──────────────┘
       ▼
┌─────────────┐
│   Agent     │  规划、多步、工具调用、记忆
└──────┬──────┘
       ▼
┌─────────────┐     ┌──────────────┐
│   Skill     │────▶│  Rules       │  可复用指令；.cursorrules 常驻规则
└──────┬──────┘     └──────────────┘
       ▼
┌─────────────┐
│    MCP      │  标准化连接 Obsidian、Git、浏览器、API…
└──────┬──────┘
       ▼
┌─────────────┐
│ 工作流编排   │  模板、plan、续做、复盘 ← 你的 AI-Work-Kit
└─────────────┘
```

## 易混概念对照

| 概念 A | 概念 B | 区别 |
|--------|--------|------|
| Prompt | Skill | Prompt 是一次性输入；Skill 是固化、可反复调用的 prompt + 流程 |
| Skill | MCP | Skill 教 AI **怎么做**；MCP 给 AI **能摸到什么数据/工具** |
| RAG | MCP 搜索 | RAG 泛指检索增强生成；MCP 是一种让 Cursor 调检索工具的方式 |
| Agent | Chat | Chat 通常一问一答；Agent 可自主多步、调工具 |
| .cursorrules | Skill | rules 始终生效；Skill 按任务类型触发 |
| Obsidian `Skills/` | `.cursor/skills/` | 前者给人看+`@`；后者给 Cursor Agent 自动读 |
| Plan 续做 | 学习续学 | 同一套「进度+下一步」，场景不同 |
