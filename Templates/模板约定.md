# 模板约定（全库统一）

## 日期

- 占位符统一 **`{{date}}`**（Obsidian 全局格式见 `.obsidian/templates.json`，推荐 `YYYY-MM-DD`）。
- 文件名：`{{date}}-{{title}}.md`。

## Plan 状态（YAML + 正文一致）

```yaml
status: 草稿 | 进行中 | 评审中 | 已采纳 | 搁置
```

子类型可追加说明（如需求分析「待产品确认」写在正文，顶层仍用上述枚举）。

## 续做（全库唯一格式）

```
/resume plan=Plans/【分类】/{{date}}-{{title}}.md 进度=【当前完成情况】
```

由 `resume-assistant` 解析；**不要**在 plan 里写 `/feature-dev-assistant 续做` 等变体。

## 标签

统一 YAML 数组：`tags: [排查, 模板]`。

标准集：`排查` `方案` `功能开发` `需求分析` `figma` `日报` `周报` `学习` `review` `会议` `复盘` `api` `发布` `资料`

| 模板 | 用途 |
|------|------|
| `客户端功能开发模板.md` | 功能开发（含/不含业务逻辑） |

| `资料沉淀模板.md` | PM 物料 / Contexts 配置对照表（配合 `/material-prep`） |

已删除旧模板：`功能开发模板`、`Figma界面开发模板`、`客户端/服务端技术方案模板` → 分别用 `客户端功能开发模板`、`技术方案模板`。

## 存放路径

| 类型 | 路径 |
|------|------|
| Plan | `Plans/【中文分类】/{{date}}-{{title}}.md` |
| 知识 / PM 物料 | `Contexts/【主题】/`（如 `Contexts/收银台/`） |
| 日报 | `Contexts/日报/{{date}}.md` |
| 周报 | `Contexts/周报/{{date}}至{{date}}.md` |

## 章节编号

中文文档用 **一、二、三…**（从一起），不用「零、」或 `0.`。
