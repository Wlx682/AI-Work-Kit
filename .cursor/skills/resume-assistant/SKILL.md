# 续做助手

当用户说"续做"并提供 `plan文件名`（含子目录，如 `Bug排查/xxx.md`）和 `当前进度` 时：

## 步骤

1. 读取 `Plans/[plan文件名]`；若在根目录未找到，搜索 `Plans/**/[文件名]`。
2. 读取 plan 中 `[[双向链接]]` 引用的 `Contexts/` 笔记。
3. 根据当前进度判断下一步。
4. 输出：已完成 / 下一步（1-3 个动作）/ 可能原因 / 验证方法 / 风险前提。
5. 进度不足时要求补充（如"第 3 步遇到什么错误？"）。

## 输出格式

```markdown
## 续做：[任务标题]
**Plan**：`[路径]`
**当前进度**：[描述]

### 已完成
### 下一步
### 可能原因（如适用）
### 验证方法
### 需要补充（如有）
```
