## 1. 核心逻辑（PRD 原文落地）

### 1.1 §2.1 会员校验（主交付）

| 用户类型 | 会员校验 | 说明 |
|----------|----------|------|
| **绑定盒子的用户** | **全局不校验** | 使用任何功能均不校验会员状态 |
| 未绑定盒子的用户 | 按原有逻辑 | 保持现有行为 |

**设计原理**：盒子为已购硬件，绑定 ≡ 会员身份，全局解锁。

**客户端语义**：

```text
hasPremiumAccess = isValidMember（订阅） OR hasBoundBox（绑定盒子）

功能门槛（Guard / require* / 上传前校验 / 连云虾）→ hasPremiumAccess
收银台 / 到期日 / 续费文案 → 仍用 isValidMember（订阅语义）
```

### 1.2 §2.2 龙虾类型切换（联调）

| 用户类型 | 切换行为 |
|----------|----------|
| **绑定盒子的用户** | 云龙虾 ↔ 盒子龙虾 **自由切换，无任何拦截** |
| 未绑定盒子的用户 | **仅可使用云龙虾** |

**代码对照**：

| 点 | 现状 | 动作 |
|----|------|------|
| 切换入口可见性 | `hardware_group(showCurrentDevice: clawCount > 1)` | PRD 称当前已满足，**不改** |
| 连云虾 | `NMClawConnector.requireVipForConnectCloud` 校验会员 | 改为 `hasPremiumAccess`；盒子用户直接过 |
| 连盒子虾 | `connectBoxClaw` 不校验会员 | 保持 |
| 非盒子用户见盒子虾 | `fetchMyClawList` 返回 cloud+box | 依赖**服务端**不返回未绑定盒子；客户端可加防御过滤 |

### 1.3 §2.3 文件上传路由（联调）

| 当前龙虾类型 | 上传目标 | 云盘类型 |
|--------------|----------|----------|
| 云龙虾 | 云盘 | 主云盘（原有） |
| **盒子龙虾** | **云盘** | 主云盘（原有） |

**代码对照**：

- 对话 InputBar 已用 `NMCloudFileUploadHelper`（`NMFileUploadListView`）→ 云盘通道 ✅
- 上传前仍走 `NMCloudUploader.handleCloudMember` → `isValidMember` ❌ 需改为 `hasPremiumAccess`
- 需排查盒子虾场景是否仍有七牛/CDN 直传分支（`NMFileUploadManager` 遗留路径）

### 1.4 §2.4 云盘配额（云盘模块 + 服务端）

| 云盘类型 | 容量 | 配额来源 |
|----------|------|----------|
| 主云盘 | 现有逻辑 | 服务端 |
| **盒子专属小云盘** | 待定 | 服务端定义 |

- **盒子所有者 + 非会员**：单独配额；解绑后取消
- **本次不做**：配额展示与升级入口（PRD 四）

## 3. 技术方案（会员主交付）

### 3.1 服务层扩展

1. **`NMMembership` 增字段**（优先服务端 `api/membership/info`）  
   - 例：`has_bound_box: Bool`

2. **`NMMembershipService` 新 API**

```swift
/// 是否跳过会员校验（订阅会员 OR 已绑定盒子）
func hasPremiumAccess() -> Bool

/// 仅盒子权益、无订阅
func isBoxOnlyPremium() -> Bool
```

3. **改造拦截入口**（不改 30+ 散落调用，集中改 Guard / require*）

| API | 改造 |
|-----|------|
| `NMMembershipGuard.isVip` / `checkVip` | → `hasPremiumAccess` |
| `requireValidMember` / `requireCloudDriveMember` | → `hasPremiumAccess` |
| `isValidMember()` | **保持订阅语义** |

4. **`isFunctionAvailable`**：盒子用户若仍走权益字典，需 `hasPremiumAccess` 短路返回 true（PRD：全局不校验）

### 3.2 状态同步

| 事件 | 动作 |
|------|------|
| 登录 / `startupSync` | 拉 `membership/info` |
| 盒子绑定成功 | 刷新 membership |
| 盒子解绑成功 | 刷新 membership + `claw/my` |
| `NMMembershipDidUpdate` | 依赖方自动更新 |

### 3.3 客户端兜底（P0-1 服务端未就绪时）

```text
NMBoxBindingStore.hasBoundBox = claw/my 存在 type==2
hasPremiumAccess = isValidMember || hasBoundBox
```

## 6. 验收标准

1. **绑定盒子 + 非订阅**：全功能不弹收银台；云虾↔盒子虾自由切换
2. **未绑定盒子 + 非会员**：与现网一致；不可使用盒子虾（列表/连接）
3. **解绑后**（无订阅）：会员拦截恢复；盒子虾消失；小云盘收回（云盘验收）
4. **订阅会员**：不受盒子状态负面影响
5. **盒子离线**：切换入口仍可见；连接走现有离线逻辑
6. **盒子虾对话上传**：走云盘通道，不因非会员失败
