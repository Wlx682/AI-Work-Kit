## 1. 核心逻辑

### 1.1 §2.1 会员校验（主交付）

| 用户类型 | 会员校验 | 说明 |
|----------|----------|------|
| **绑定盒子的用户** | **全局不校验** | 使用任何功能均不校验会员状态 |
| 未绑定盒子的用户 | 按原有逻辑 | 保持现有行为 |

**客户端语义（2026-06-11 产品澄清）**：

```text
hasBoundBox = api/claw/my 存在 type == 2
isUsingBoxClaw = 当前操控龙虾 type == .box

云盘：无单独「云盘会员」；`canAccessCloudDrive` = 订阅会员 OR 已绑定盒子（纯云虾非会员拦截 Tab）
云龙虾：必须 isValidMember（订阅会员）
盒子龙虾：hasBoundBox 即可，无需订阅

hasPremiumAccess = isValidMember OR (hasBoundBox && isUsingBoxClaw)
功能门槛（Guard / require* / 对话上传等）→ hasPremiumAccess
连云虾（NMClawConnector）→ 仅 isValidMember
收银台 / 到期日 / 续费文案 → isValidMember
```

**典型路径**：买盒子 → 装 App → 首页可能提示购会员 → 我的页绑定盒子 → 切换到盒子虾 → 免会员使用。

### 1.2 §2.2 龙虾类型切换（联调）

| 用户类型 | 切换行为 |
|----------|----------|
| **绑定盒子的用户** | 云龙虾 ↔ 盒子龙虾 **自由切换，无任何拦截** |
| 未绑定盒子的用户 | **仅可使用云龙虾** |

| 点 | 动作 |
|----|------|
| 切换入口可见性 | PRD 当前已满足，**不改** |
| 连云虾 | `requireVipForConnectCloud` → `hasPremiumAccess` |
| 非盒子用户见盒子虾 | 依赖服务端 `claw/my` 不返回未绑定盒子 |

### 1.3 §2.3 文件上传路由（联调）

- 云虾 / 盒子虾对话上传均走 **云盘**（`NMCloudFileUploadHelper`）
- `NMCloudUploader.handleCloudMember` → `hasPremiumAccess`

## 3. 技术方案

### 3.1 `NMBoxBindingStore`（或等价模块）

- 缓存 `hasBoundBox`，数据来源 **`NMClawRequest.fetchMyClawList`**
- 判定：`list.contains { $0.type == .box }`
- 刷新时机：登录 / 冷启动、`startupSync` 后、**绑定成功**、**解绑成功**

### 3.2 `NMMembershipService`

```swift
func hasBoundBox() -> Bool          // 读 BoxBindingStore
func hasPremiumAccess() -> Bool     // isValidMember || hasBoundBox()
func isBoxOnlyPremium() -> Bool    // hasBoundBox && !isValidMember
```

- **不改** `NMMembership` 模型、**不依赖** `membership/info` 新字段

### 3.3 拦截改造

| API | 改造 |
|-----|------|
| `NMMembershipGuard.isVip` / `checkVip` | → `hasPremiumAccess` |
| `requireValidMember` / `requireCloudDriveMember` | → `hasPremiumAccess` |
| `isFunctionAvailable` | `hasPremiumAccess` 时短路 `true` |
| `isValidMember()` | **保持订阅语义** |

### 3.4 绑 / 解绑刷新

| 事件 | 动作 |
|------|------|
| 盒子绑定成功 | `fetchMyClawList` 更新 `hasBoundBox` + `fetchMembership` |
| 盒子解绑成功 | **立即** `fetchMembership` + `fetchMyClawList` |

## 6. 验收标准

1. **绑定盒子 + 非订阅**：全功能不弹收银台；云虾↔盒子虾自由切换
2. **未绑定盒子 + 非会员**：与现网一致
3. **解绑后**（无订阅）：刷接口后会员拦截恢复；列表无盒子虾
4. **订阅会员**：不受盒子状态负面影响
5. **盒子虾对话上传**：走云盘，不因非会员失败
