## 二、背景与目标

### 2.1 新用户首次启动（图 1）

- **业务需求**：登录后全屏加载页阻塞首页；后台静默创建/连接云龙虾；成功自动进首页可对话
- **失败分流（产品澄清）**：
  - **未开通 / 会员过期**导致无法继续 → dismiss 加载页 → **进首页** → 弹 **§2.2 会员弹窗**（过期 / 未开通文案）
  - **其他错误**（网络、创建主机失败、连接失败等）→ **留在加载页**，展示重试 / 联系客服（**不进首页**）
- **成功指标**：新用户从登录到可发首条消息的路径 ≤ 1 次用户操作（等待即可）
- **非目标**：改造 `NMCloudClawDeployView` 手动「立即体验」引导页（新用户不应再看到）；技术失败时 dismiss 到首页再弹「连接失败」

### 2.2 会员过期处理（图 2）

- **业务需求**：APP 启动（已登录）检测会员 → 有效则静默；过期/未开通则先进首页再轻量弹窗
- **成功指标**：有效会员 0 打扰；过期/未开通用户 100% 在首页看到区分文案的弹窗
- **非目标**：全屏拦截、强制跳转收银台

### 2.3 服务拆分

- **业务需求**：纳米 Work 会员服务与安全龙虾侧 **后端独立部署**；客户端接口尽量不变；算力豆数值 ×1000
- **成功指标**：客户端无感知切换（或按约定适配）；余额/扣费展示与后端一致
- **非目标**：客户端拆分两套会员 SDK

## 0. 当前 vs 目标（代码对照）

### 0.1 §2.1 新用户首次启动

| 维度 | 当前实现 | 目标 |
|------|----------|------|
| 触发时机 | 进首页 `viewDidAppear` 后 present `NMPrepareChatController` | **登录后、进首页前/覆盖首页** 全屏加载 |
| 无设备处理 | 列表空 → dismiss PrepareChat → push 聊天页 → 用户需走 `allocateVM()` | **后台自动** `allocateVM` + 轮询 + 连接 |
| 会员门槛 | `allocateVM()` 先 `fetchMembership`，非 VIP 弹收银台 | 加载页内校验；**未开通/过期** → 进首页 + §2.2 会员弹窗 |
| 加载文案 | `NMClawConnectStepView`：连接/龙虾升级中/扩容 | 按操作类型：**正在连接 / 专家升级中 / 正在扩容云主机** |
| 失败处理 | Toast + 聊天页 `NMRetryView` | **会员原因**：进首页 + 会员弹窗；**其他错误**：**停留加载页** + `NMRetryView`（重试/联系客服） |
| 成功处理 | 连接成功 dismiss PrepareChat，用户已在首页底层 | 加载页消失，**首页完全可用** |

#### §2.1 失败分流（产品口径）

| 失败类型 | 页面 | UI | 用户操作 |
|----------|------|-----|----------|
| 未开通会员 / 会员已过期 | 首页（先 dismiss 加载页） | §2.2 会员弹窗（区分过期 vs 未开通） | 稍后再说 / 立即开通 |
| 拉列表失败、创建主机失败、连接失败、轮询超时等 | **加载页** | `NMRetryView` 或同等错误态（可含「连接失败，稍后重试」类文案） | 重试 / 联系客服 |

> PRD 流程图「创建失败 → 进首页」指 **会员不可用** 分支，非所有技术失败。

**关键代码位置**：

- `NMHomeShellViewController.bootstrapGatewayConnectionIfNeeded` — 首页才触发
- `NMPrepareChatController.handleEmptyClawList` — 空列表转创建，非静默
- `NMChatViewController+Claw.allocateVM` — 含会员校验 + Toast 失败
- `NMClawConnectStepView` — 文案映射（`clawUpgrade` → 需改「专家升级中」）
- `NMClawVMExpansionTrigger` — 扩容文案「正在扩容主机」→「正在扩容云主机」

### 0.2 §2.2 会员过期处理

| 维度 | 当前实现 | 目标 |
|------|----------|------|
| 触发时机 | 仅 `NMGatewayLaunchBootstrap.showPostConnectPopups`（Gateway 连接完成后） | **APP 启动**（已登录进 Tab）即检测 |
| 有效会员 | `isVip()` → 不弹 | 一致 |
| 过期文案 | 统一「会员已到期」+「360安全龙虾」（安全龙虾遗留文案） | **过期**：「会员已过期，续费后可继续使用，历史数据保留15天」 |
| 未开通文案 | 同上（未区分） | **未开通**：「开通纳米Work会员，解锁全部功能」 |
| 按钮 | 「稍后再说」/「立即续费」 | 「稍后再说」/「立即开通」（PRD 表；正文有一处写「立即续费」，以表为准待产品拍板） |
| 收银台 | `NMPaymentManager.openMember` | 一致 |
| 关闭后 | 停留当前页 | 停留首页，「我的」可管理账号 |

**关键代码位置**：

- `NMMembershipGuard.showExpirationPopupIfNeeded` — 需拆分 `isExpireMember` / `nonMember` 文案
- `NMVipView.resolveState` — 已有过期/未开通状态分辨逻辑，可复用
- `NMMembershipService+Sync.syncAllDataSilently` — 登录后拉会员，可作为弹窗数据源

## 四、架构概要

| 模块/类 | 职责 |
|---------|------|
| **`NMCloudClawBootstrapCoordinator`**（新建） | 登录后/首页前编排：拉设备列表 → 空则创建 → 轮询 → 连接；对外暴露状态机 |
| **`NMPrepareChatController`**（扩展） | 全屏加载 UI；接收 Coordinator 状态；映射加载文案 |
| **`NMGatewayLaunchBootstrap`**（调整） | 会员弹窗职责迁出；仅保留 Gateway 连接编排 |
| **`NMMainTabBarController` / `NMHomeShellViewController`** | 启动时触发 Coordinator；成功后展示首页 |
| **`NMMembershipLaunchGuard`**（新建或扩展 Guard） | 启动会员检测 + 区分过期/未开通弹窗 |
| **`NMMembershipGuard`** | 功能点拦截保持不变 |
| **`NMClawConnectStepView`** | 文案常量对齐 PRD |
| **`NMCoinBalance` / 展示层** | §2.3 数值适配（若 P0-5 为客户端职责） |

### 状态机（§2.1）

```text
idle → fetchingList → [hasClaw] connecting → success → dismiss（进首页）
                   → [empty] creating → polling → connecting → success
                   → membershipFail → dismiss → home + §2.2 会员弹窗
                   → technicalFail → 停留加载页 + RetryView（重试/联系客服）
```

### API

- 现有：`api/user/vm/create_host`、`claw/my`、VM status 轮询 — **不变**
- §2.3：待服务端确认 `membership/info`、`coin/balance` 字段语义

### 特性开关

- 建议云控 `cloud_claw_auto_bootstrap` 便于灰度

## 六、验收

### §2.1

- [ ] 新用户登录后**不进入可操作首页**，直至云龙虾就绪，或遇会员/技术失败进入对应态
- [ ] 三种操作类型展示对应加载文案
- [ ] 创建成功：自动进首页，输入框可发消息
- [ ] **会员未开通/过期**：进首页 + §2.2 会员弹窗（非「连接失败」文案）
- [ ] **技术失败**：停留加载页；重试可重新走流程；联系客服可跳转（路由待确认）

### §2.2

- [ ] 有效会员冷启动：无会员弹窗
- [ ] 过期会员：首页弹窗 + 正确过期文案 + 15 天保留说明
- [ ] 未开通：首页弹窗 + 未开通文案
- [ ] 「稍后再说」关闭后停留首页（功能受限逻辑保持现有 Guard）
- [ ] 「立即开通」跳转会员收银台

### §2.3

- [ ] 会员/余额接口切换后功能正常
- [ ] 算力豆展示、扣费、不足提示与后端一致
