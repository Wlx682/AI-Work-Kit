## 二、背景与目标

### 业务需求：「非会员关闭弹窗后的功能受限」

用户在首页（或其它入口）对会员提示弹窗点 **「稍后再说」** 后：

| Tab | 索引 | 非有效会员行为 |
|-----|------|----------------|
| **首页** | 0 | ✅ 可访问 |
| **任务** | 1 | ❌ 拦截，弹 **会员过期/未开通弹窗**（同首页 `NMMembershipExpirationPopupController`） |
| **专家** | 2 | ❌ 同上 |
| **云盘** | 3 | ❌ 同上 |
| **我的** | 4 | ✅ 可访问（账号管理等） |

- **弹窗类型**：`showExpirationPopupIfNeeded`（区分过期 / 未开通文案），**非**直接 `presentPaywall` 收银台
- **拦截方式**：`shouldSelect` 返回 `false`，停留在当前 Tab

### 成功指标

- 非会员点受限 Tab 100% 弹出会员提示窗，且无法进入该 Tab
- 首页、我的始终可切换
- 有效会员行为与改前一致（不受本规则影响）

### 非目标

- 首页内输入框 / 发消息等细粒度拦截（沿用现有 `NMMembershipGuard.checkVip` 等）
- 修改启动 / PrepareChat 流程
- 安全龙虾 App 行为

## 四、架构概要

| 改动点 | 职责 |
|--------|------|
| `NMMainTabBarController.shouldSelect` | 受限 Tab 集合 `{1,2,3}`；非 VIP 弹窗并 `return false` |
| `NMMembershipGuard` | 可选：抽 `static func shouldAllowTabSwitch(index:) -> Bool` 或 `interceptRestrictedTab(from:)`，避免 TabBar 内堆逻辑 |
| `NMMembershipExpirationPopupController` | 无改动（已支持文案入参） |

### 推荐实现草图

```swift
private static let membershipRestrictedTabIndexes: Set<Int> = [1, 2, 3]

private func shouldSelect(vc: UIViewController) -> Bool {
    guard let index = viewControllers?.firstIndex(of: vc) else { return false }
    guard NMMembershipGuard.isVip() else {
        if Self.membershipRestrictedTabIndexes.contains(index) {
            NMMembershipGuard.showExpirationPopupIfNeeded(
                from: self,
                reshowAfterPaywallClose: true
            )
            return false
        }
        return true
    }
    // 有效会员：保留 NMCloudClawDeployGuard 逻辑
    ...
}
```

**注意**：`showExpirationPopupIfNeeded` 需有效 presenter；`from: self`（TabBar）经 `topPresenter` 链应可达。
