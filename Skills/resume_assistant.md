# 续做助手 Skill

当用户说"续做"并提供 `plan文件名` 和 `当前进度` 时，请执行以下步骤：

## 执行步骤

1. 读取 `Plans/[plan文件名]` 笔记（含子目录，如 `Bug排查/2026-06-10-API超时排查.md`）。
2. 若 plan 中引用了 `Contexts/` 材料，一并读取相关上下文。
3. 根据当前进度，判断下一步该做什么。
4. 输出结构化结果：
   - **已完成**：根据用户进度 + plan 内容归纳
   - **下一步具体操作**：可执行的 1-3 个动作
   - **可能的原因**（排查类任务）
   - **验证方法**：如何确认这一步做对了
   - **风险/前提**：plan 中写明的约束条件
5. 如果进度信息不足，要求用户补充（例如"已完成步骤 1 和 2，在第 3 步遇到了什么错误？"）。

## 输出格式

```markdown
## 续做：[任务标题]

**Plan**：`[文件名]`
**当前进度**：[用户描述]

### 已完成
- ...

### 下一步
1. ...

### 可能原因（如适用）
- ...

### 验证方法
- ...

### 需要补充的信息（如有）
- ...
```

## 触发示例

```
@Skills/resume_assistant.md 续做，plan文件名=Bug排查/2026-06-10-API超时排查.md，当前进度=已完成第1步抓包，第2步分析到一半
```
