# Figma 界面开发 Skill

当用户说「做界面」「Figma 开发」「还原设计稿」「UI 实现」时执行。

> **定位：仅 UI 子任务。** 新功能/新模块（方案+界面一起）→ 用 `feature-dev-assistant`，不要从本 Skill 开工。

## 知识库路径

- 公共规范：`Contexts/Figma/项目设计规范.md`
- 最佳实践：`Contexts/Figma/Figma界面开发最佳实践.md`
- 模板：`Templates/Figma界面开发模板.md`

**代码仓库**：默认 = 当前 Cursor 工作区；工作区是 AI-Work-Kit 时加 `仓库=/path` 或先打开代码项目。
- Plan 存放：`Plans/界面开发/`

## 新任务（开工）

1. 读取项目设计规范 + 最佳实践（**不读、不猜 Figma URL**）。
2. 若用户未提供 Figma 链接，先索要；链接只用于当次任务和 plan，不写回 Contexts。
3. 读取 `Templates/Figma界面开发模板.md`，填充：链接、Frame 名、平台、需求摘要。
4. 在**代码仓库**内搜索类似 UI 组件；不查知识库功能清单（已开发完的不沉淀）。
5. 有 Figma 链接时：**用 Figma MCP 逐节点读取** Frame 及子 layer（尺寸、圆角、描边、阴影、fill、字体、间距、Variant），**禁止仅凭截图估像素**。
6. 输出：
   - **节点度量表**（@2x px → iOS pt，注明来源 layer 名）
   - **组件映射表**（复用 / 扩展 / 新建）
   - **状态/Variant 清单**（默认、按压等；无稿标「待设计确认」）
   - **兜底项说明**（仅列出设计未标注、将用规范默认值的属性）
   - **Plan 草稿** + 建议保存路径
   - **第一步指令**（通常从读节点 + 布局骨架开始）
7. 用户确认后再写代码。

## 续做

用户提供 `plan文件名` + `当前进度` 时：

1. 读取 `Plans/界面开发/[plan文件名]`
2. 对照实施步骤勾选，从下一未完成的切片继续
3. 输出：已完成 / 本步实现要点 / 下一步 / 待设计确认项

## 设计走查

用户说「走查」「对稿」时：

1. 读取 `Templates/Figma设计走查模板.md`
2. 对照 plan 中的 token 和组件映射逐项检查
3. 输出差异表 + 处理建议

## 触发示例

```
/figma-ui-assistant 新任务，Figma=【你粘贴的链接】，平台=iOS，页面=0608模型选择

/figma-ui-assistant 续做，plan=界面开发/2026-06-10-设置页.md，进度=已完成静态UI

/figma-ui-assistant 走查，plan=界面开发/2026-06-10-设置页.md
```

## 代码原则

- **1:1 还原**：数值来自 Figma 节点（@2x px ÷ 2 = pt）；规范默认值仅设计未标注时使用
- **逐节点读取**，不靠截图目测；按压/禁用等读 Variant
- 先复用现有组件，再按读出尺寸实现
- 切片实施：读节点 → 骨架 → 静态 → 数据 → 交互 → 异常态 → 走查
- 不猜测未确认的状态稿；无按压稿须在 plan 标明
