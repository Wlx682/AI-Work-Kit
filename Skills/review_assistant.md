# 复盘助理 Skill

当用户说"复盘"并提供时间段（或本月主要任务列表）时，生成结构化复盘报告。

## 执行步骤

1. 搜索 `Plans/` 和 `Contexts/` 中该时间段内的笔记（按文件名日期或内容）。
2. 若用户提供了任务清单，与知识库材料交叉核对。
3. 按"顺利任务 / 踩坑任务"分类，并标注是否使用了 AI、模板、plan 续做。
4. 统计可量化数据（用户提供的或从笔记推断）：
   - 续做次数与节省的背景交代时间
   - 各模板使用次数
   - 材料复用次数
5. 输出结论：适合 AI 的场景 + 需要谨慎的场景 + 至少 3 条改进行动。

## 输出格式

```markdown
# 复盘报告：[时间段]

## 概览
| 指标 | 数值 |
|------|------|
| 续做成功率 | x/y |
| 模板使用 | 排查 n 次，方案 n 次，review n 次 |
| 材料复用 | n 次 |

## 顺利任务（AI 帮上忙）
| 任务 | 用了什么 | 效果 |
|------|----------|------|

## 踩坑任务（AI 添乱或无效）
| 任务 | 问题 | 教训 |
|------|------|------|

## 适合继续用 AI 的场景
1. ...

## 需要谨慎或人工主导的场景
1. ...

## 改进行动（至少 3 条）
1. ...
2. ...
3. ...

## 下月目标
- ...
```

## 触发示例

```
@Skills/review_assistant.md 复盘时间段=2026-06-01 至 2026-06-30
```

若知识库中该时段材料不足，请用户手动列出 5-10 个主要任务并标注 AI 使用情况。
