# 学习助手 Skill

当用户说「学习」「续学」「考我」「整理学习笔记」「新概念」时执行。

## 必读

- [[Contexts/LLM学习/学习路线-LLM与提示词]]
- [[Contexts/LLM学习/知识地图]]
- `Templates/学习笔记模板.md`、`Templates/概念卡模板.md`

## 模式

### 1. 新主题

用户提供：题目、学习目标、可选资料链接。

1. 查 `Contexts/LLM学习/` 是否已有相关概念卡。
2. 输出：学习计划（3–5 步）+ 建议 plan 路径 `Plans/学习/YYYY-MM-DD-主题.md`。
3. 第一步阅读/实践建议；关联 Kit 中的例子。

### 2. 续学

用户提供：`plan文件名` + `当前进度`。

1. 读 `Plans/学习/[plan]`。
2. 输出：已完成 / 下一步 / 推荐笔记模板 / 自测题 1–2 道。

### 3. 整理笔记

用户粘贴草稿或要点。

1. 按 [[Templates/学习笔记模板]] 结构化。
2. 建议是否升级为概念卡（`Contexts/LLM学习/概念/`）。
3. 补充与 Kit（Skill/MCP/模板）的关联。

### 4. 考我（费曼）

用户给范围（如「本周 Agent 和 MCP」）。

1. 基于 `Contexts/LLM学习/` 出题：概念题 + 让用 AI-Work-Kit 举例。
2. 用户答后给反馈；薄弱点建议回看哪张概念卡。

## 输出原则

- 先引用知识库已有概念卡，避免重复造概念。
- 外链和课程链接不写入 Contexts，除非用户要求归档。
- 鼓励「学完就写概念卡」，plan 可删。

## 学习顺序（勿跳课）

1. LLM + 提示词 → 2. 上下文 → 3. RAG → 4. Agent → 5. Skill → **6. MCP**

新用户默认从 `Plans/学习/2026-06-12-第1课-LLM与提示词入门.md` 开始；用户要求 MCP 但未到第 6 课 → 提醒先完成前置。

## 触发示例

```
/learn-assistant 新主题，题目=LLM基础与提示词，目标=能写结构化prompt

/learn-assistant 续学，plan=学习/2026-06-12-第1课-LLM与提示词入门.md，进度=读完概念卡还没练手

/learn-assistant 整理笔记，主题=提示词结构，要点=角色+任务+输出格式...

/learn-assistant 考我，范围=LLM基础和提示词（第1课）
```
