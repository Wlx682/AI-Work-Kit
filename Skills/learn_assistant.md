# 学习助手 Skill

当用户说「学习」「续学」「考我」「整理学习笔记」「新概念」或 **任何学习相关** 请求时执行。

> **进度采集与动态出资料均在本 Skill 内完成**；全量交叉审计转 [[learning_audit_assistant]]。

## 必读

- [[Contexts/LLM学习/学习路线-LLM与提示词]]
- [[Contexts/LLM学习/知识地图]]
- `Templates/学习笔记模板.md`、`Templates/概念卡模板.md`

---

## 统一会话协议（所有学习相关触发必守）

**不论** 新主题 / 续学 / 整理笔记 / 考我 / 用户随口问一个 LLM 概念 —— 只要走 `learn-assistant`，都执行：

```
开场读进度 → 按进度动态出资料 → 会话内容 → 收尾自动收集
```

| 阶段 | 必做 | 脚本 / 资料 |
|------|------|-------------|
| **1. 开场读进度** | 先读再讲 | `./scripts/learning-progress-read.sh` |
| **2. 动态出资料** | 见下文「动态出资料规则」 | plan + 快照 + 审计报告 + 概念卡 |
| **3. 模式执行** | 新主题 / 续学 / 整理笔记 / 考我 | 各模式步骤 |
| **4. 收尾收集** | 会话结束前必做 | `./scripts/learning-progress-snapshot.sh` |

审计专用请求（「审计学习进度」「learning-audit」）→ 仍先 **开场读进度**，再转 [[learning_audit_assistant]]，审计结束后也 **收尾快照**（mode=审计）。

## 动态出资料规则（续学 / 再次学习时核心）

根据开场读到的证据，**只出下一步该看的**，不从头重复：

| 证据 | 本次侧重 |
|------|----------|
| plan 有未勾选步骤 | 只讲 **第一个** `- [ ]` 对应内容；附带概念卡链接 |
| 步骤全勾、无本课笔记 | 引导 **整理笔记** 或 **练习 A/B**（未填的表） |
| 练习未完成 | 只出题 / 批改练习，不讲新课 |
| 快照/审计显示「严重矛盾」 | **不信** frontmatter「已完成」；按勾选从断点续 |
| 用户说「复习」 | 从快照里 **薄弱点** + 考我错题；不新开章节 |
| 新用户、无 plan | 按 [[学习路线-LLM与提示词]] 第 1 课开工 |
| 要求跳课（如 MCP） | 对照路线前置是否完成；未完成则建议先补 |

**输出结构建议**（续学 / 再次学习）：

```markdown
### 你的进度（自动读取）
- 当前课：…
- 步骤：x/y 已勾选
- 笔记：有/无
- 本次聚焦：【一步】

### 本次资料
（只给与「本次聚焦」相关的内容）

### 自测 1 题
```

## 模式

### 1. 新主题

用户提供：题目、学习目标、可选资料链接。

1. **开场读进度**（统一协议）。
2. 若路线中已有同主题 plan 且未完成 → **改走续学**，不新建重复 plan。
3. 查 `Contexts/LLM学习/` 是否已有相关概念卡。
4. 输出：学习计划（3–5 步）+ 建议 plan 路径 `Plans/学习/YYYY-MM-DD-主题.md`。
5. **收尾自动收集**。

### 2. 续学

用户提供：`plan文件名` + `当前进度`（可省略，由开场脚本推断）。

1. **开场读进度** + 读 `Plans/学习/[plan]`。
2. **动态出资料**：按上表只出下一未勾步骤 + 自测 1–2 道。
3. 用户确认完成的步骤 → 勾选 `- [x]`。
4. **收尾自动收集**。

### 3. 整理笔记

用户粘贴草稿或要点。

1. **开场读进度**。
2. 按 [[Templates/学习笔记模板]] 结构化；对齐当前课未勾步骤。
3. 建议是否升级为概念卡。
4. **收尾自动收集**。

### 4. 考我（费曼）

用户给范围（如「本周 Agent」）。

1. **开场读进度**；优先考 **未勾步骤 / 审计标不一致** 的课。
2. 出题 + 反馈薄弱点。
3. **收尾自动收集**（summary 记薄弱点，供下次动态出题）。

### 5. 审计进度

1. **开场读进度**。
2. 转 [[learning_audit_assistant]] 全量审计。
3. **收尾快照**（mode=审计）。

## 学习收尾 · 自动收集（所有模式必做）

会话**结束前**自动执行，无需用户再说「收集一下」：

1. 用户确认完成的步骤 → 勾选 plan `- [x]`。
2. ```bash
   ./scripts/learning-progress-snapshot.sh --mode <模式> --plan <plan路径> --summary "<一句摘要>"
   ```
   `--mode`：`新主题` / `续学` / `整理笔记` / `考我` / `审计`。
3. 回复末尾附 **「本次进度」** 小表（勾选、笔记、与 status 是否一致）。
4. 矛盾明显 → 提醒 `/learning-audit-assistant`。

快照：`Contexts/LLM学习/笔记/学习进度快照.md`  
全量审计：[[learning_audit_assistant]]（可周末 Automations）

## 写在哪里？（分工）

| 内容 | 放哪 | 原因 |
|------|------|------|
| 开场读进度、动态出资料、收尾收集 | **本 Skill**（`learn_assistant`） | 学习会话编排，每次触发都要跑 |
| 全量交叉审计逻辑 | [[learning_audit_assistant]] | 重、低频 |
| 机械取证脚本 | `scripts/learning-audit-collect.sh` | 确定性 |
| 开场摘要脚本 | `scripts/learning-progress-read.sh` | 确定性 |
| 快照追加 | `scripts/learning-progress-snapshot.sh` | 确定性 |
| 路线与概念 | `Contexts/LLM学习/` | 长期资料 |
| Cursor 自动匹配 | `.cursor/skills/learn-assistant/SKILL.md` | 与本文同步 |

**不要**把动态出资料写进 `.cursorrules` 细则 —— 只保留一句「学习相关走 learn-assistant」；细节在本 Skill。

## 输出原则

- 先引用知识库已有概念卡，避免重复造概念。
- 外链和课程链接不写入 Contexts，除非用户要求归档。
- 鼓励「学完就写概念卡」，plan 可删。

## 学习顺序（勿跳课）

1. LLM + 提示词 → 2. 上下文 → 3. RAG → 4. Agent → 5. Skill → **6. MCP**
