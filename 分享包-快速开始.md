# AI-Work-Kit 快速开始（5 分钟版）

> 给收到分享包的同事 · 完整方案见 [[分享方案-AI知识库与Cursor协作]] · 模板约定 [[Templates/模板约定]]

## 1. 打开（1 分钟）

| 工具 | 操作 |
|------|------|
| Obsidian | `Open folder as vault` → 选 `AI-Work-Kit` |
| Cursor | `Open Folder` → 同一个 `AI-Work-Kit`（**或在业务代码项目里用 `/skill`**） |

## 2. MCP（可选）

改 `.cursor/mcp.json` 里 `--vault` 为你的绝对路径 → `Reload Window` → enquire 绿灯。

不用 MCP 也能用：`/skill` 或 `@Skills/xxx.md` 足够。详见 [[Contexts/MCP进阶指南]]。

---

## 3. 怎么说（一句话版）

| 任务              | 在代码项目里（推荐）                                                                    |
| --------------- | ----------------------------------------------------------------------------- |
| 排查 bug          | `/template-generator 任务类型=排查，背景=...`                                          |
| **续做**          | `/resume plan=Plans/Bug排查/xxx.md 进度=...`                                      |
| 技术方案            | `/template-generator 任务类型=技术方案，平台=客户端/服务端，背景=...`                             |
| Code Review     | `/template-generator 任务类型=review，背景=...`                                      |
| **分析 PRD**      | `/requirement-analyst 新分析，PRD=【链接】，模块=【名】`                                    |
| **新功能/模块**      | `/feature-dev-assistant 新任务，模块=【名】，需求分析=Plans/需求分析/xxx.md，Figma=【链接】`         |
| 仅 UI            | `/figma-ui-assistant 新任务，Figma=【链接】，平台=iOS，页面=...`                            |
| **今日日报**        | `/review-assistant 日报`                                                        |
| **本周周报**        | `/review-assistant 周报`                                                        |
| 月底复盘            | `/review-assistant 复盘时间段=2026-06-01 至 2026-06-30`                             |
| 搜历史材料           | 「知识库里有没有关于 xxx 的排查/决策？」（要 MCP 绿灯）                                             |
| 学习 LLM          | `/learn-assistant 新主题，题目=...`（路线 [[Contexts/LLM学习/学习路线-LLM与提示词]]）             |
| **PM 资料 / 对照表** | `/material-prep 类型=收银台，参考=Claw，新App=namiWork`（写 `Contexts/`，**不写业务仓库 docs/**） |

在 **AI-Work-Kit 工作区内**把 `/xxx` 换成 `@Skills/xxx.md` 即可。

## 4. 完整 Case 参考（复制改字）

### Case 1：接到 bug，从零开始排查

```
/template-generator 任务类型=排查，背景=生产环境 /api/v1/order/create 偶发 5s 超时，约 0.3%，高峰 10-11 点；网关 5s 超时；6/8 刚上库存缓存优化；本地压测未复现
```

**存哪**：`Plans/Bug排查/2026-06-10-下单超时.md`

---

### Case 2：排查到一半，换会话续做

```
/resume plan=Plans/Bug排查/2026-06-10-下单超时.md 进度=已完成抓包，确认网关等满 5s、inventory 无响应，下一步查连接池和慢 SQL
```

---

### Case 3：写技术方案（客户端或服务端）

```
/template-generator 任务类型=技术方案，平台=客户端，背景=iOS 设置页重构，UIKit+SnapKit；VC 过重难测；目标可维护+单测；iOS 15+
```

```
/template-generator 任务类型=技术方案，平台=服务端，背景=订单查询 P99 800ms；峰值 QPS 3k；Go；评估 Redis vs 读库优化；写操作幂等
```

**存哪**：`Plans/客户端技术方案/` 或 `Plans/服务端技术方案/`（模板 [[Templates/技术方案模板]]）

---

### Case 4：需求分析 → 功能开发

```
/requirement-analyst 新分析，PRD=【飞书链接】，模块=创建自动化任务
```

P0 闭环后：

```
/feature-dev-assistant 新任务，模块=创建自动化任务，需求分析=Plans/需求分析/2026-06-10-创建自动化任务.md，Figma=【链接】
```

**存哪**：`Plans/功能开发/`（模板 [[Templates/客户端功能开发模板]]）

---

### Case 5：Code Review

```
/template-generator 任务类型=review，背景=PR #1234 分支 feature/chat-model-switch，关注线程安全和状态回显
```

---

### Case 6：Figma 还原界面（链接当次贴）

```
/figma-ui-assistant 新任务，Figma=https://www.figma.com/design/xxx?node-id=123-456，平台=iOS，页面=模型选择弹窗
```

**存哪**：`Plans/功能开发/`（含业务逻辑=否）

**走查**：

```
/figma-ui-assistant 走查，plan=Plans/功能开发/2026-06-12-模型弹窗.md
```

---

### Case 7：月底复盘

```
/review-assistant 复盘时间段=2026-06-01 至 2026-06-30
```

---

### Case 8：学 LLM / 提示词（第一课）

路线：[[Contexts/LLM学习/学习路线-LLM与提示词]]

```
/learn-assistant 新主题，题目=LLM基础与提示词入门，目标=能解释token和上下文窗口，并写出结构化prompt
```

---

### Case 9：会议结论（不经过 Skill）

Obsidian 新建：`Contexts/会议/2026-06-15-网络超时策略.md`（或用 [[Templates/会议记录模板]]）

---

### Case 10：PM 物料 / 收银台配置对照表

在 **业务代码仓库**（如 ClawAI）打开 Cursor，对照参考 App 提炼配置，沉淀到 Vault：

```
/material-prep 类型=收银台，参考=Claw，新App=namiWork
```

或：「准备资料：整理 namiWork 收银台配置，对照 Claw，写资料库」

**存哪**：`Contexts/收银台/MSPay收银台配置对照表.md`（示例；新类型用 [[Templates/资料沉淀模板]]）

**注意**：资料库 = AI-Work-Kit 的 `Contexts/`，不要写到代码项目的 `docs/`。

---

## 5. 三层记住即可

| 层 | 文件夹 | 记什么 |
|----|--------|--------|
| 模板 | `Templates/` | 每次新任务起点 |
| 规范/结论/PM 物料 | `Contexts/` | 长期有效的事实、决策、配置对照表 |
| 进行中 | `Plans/*/` | 当前任务，**做完可删** |

**不沉淀**：Figma 链接、已上线功能清单、做完的 plan。

---

## 6. 在别的项目里用

复制 `.cursor/skills/` → `~/.cursor/skills/`，在任意仓库用：

- `/template-generator` · `/resume` · `/feature-dev-assistant` · `/figma-ui-assistant` · `/review-assistant` · `/material-prep`

`@Skills/xxx.md` 只有 Cursor 打开 AI-Work-Kit 时才搜得到。
