# AI-Work-Kit 快速开始（5 分钟版）

> 给收到分享包的同事 · 完整方案见 [[分享方案-AI知识库与Cursor协作]] · 案例可复制改字即用

## 1. 打开（1 分钟）

| 工具 | 操作 |
|------|------|
| Obsidian | `Open folder as vault` → 选 `AI-Work-Kit` |
| Cursor | `Open Folder` → 同一个 `AI-Work-Kit`（**或在业务代码项目里用 `/skill`**） |

## 2. MCP（可选）

改 `.cursor/mcp.json` 里 `--vault` 为你的绝对路径 → `Reload Window` → enquire 绿灯。

不用 MCP 也能用：`/skill` 或 `@Skills/xxx.md` 足够。

---

## 3. 怎么说（一句话版）

| 任务 | 在代码项目里（推荐） |
|------|---------------------|
| 排查 bug | `/template-generator 任务类型=排查，背景=...` |
| 续做 | `/resume-assistant 续做，plan=Bug排查/xxx.md，进度=...` |
| 客户端方案 | `/template-generator 任务类型=客户端方案，背景=...` |
| 服务端方案 | `/template-generator 任务类型=服务端方案，背景=...` |
| Code Review | `/template-generator 任务类型=review，背景=...` |
| **分析 PRD**（先于开发） | `/requirement-analyst 新分析，PRD=【飞书链接】，模块=【名】` |
| **新功能/模块**（需求通过后） | `/feature-dev-assistant 新任务，模块=【名】，需求分析=需求分析/xxx.md，Figma=【链接】` |
| 仅 UI（方案已有） | `/figma-ui-assistant 新任务，Figma=【链接】，平台=iOS，页面=...` |
| 月底复盘 | `/review-assistant 复盘时间段=2026-06-01 至 2026-06-30` |
| 搜历史材料 | 「知识库里有没有关于 xxx 的排查/决策？」（要 MCP 绿灯） |
| 学习 LLM/提示词 | `/learn-assistant 新主题，题目=...，目标=...` |
| 续学 / 考我 | `/learn-assistant 续学，plan=学习/xxx.md，进度=...` |

在 **AI-Work-Kit 工作区内**把 `/xxx` 换成 `@Skills/xxx.md` 即可。

## 4. 完整 Case 参考（复制改字）

### Case 1：接到 bug，从零开始排查

**场景**：生产环境下单接口偶发 5s 超时。

**第 1 句（新会话，在 ClawAI 代码仓库里）：**

```
/template-generator 任务类型=排查，背景=生产环境 /api/v1/order/create 偶发 5s 超时，约 0.3%，高峰 10-11 点；网关 5s 超时；6/8 刚上库存缓存优化；本地压测未复现
```

**AI 应产出**：plan 草稿 → 建议存 `Plans/Bug排查/2026-06-10-下单超时.md` → 排查步骤 + 第一条命令。

**你做完第 1-2 步后存档（1 分钟）**：结论写进 plan 勾选；若有关键决策 → `Contexts/2026-06-10-库存缓存与超时.md`。

---

### Case 2：排查到一半，换会话续做

**场景**：昨天查到超时卡在 inventory，今天继续。

**新会话第 1 句：**

```
/resume-assistant 续做，plan=Bug排查/2026-06-10-下单超时.md，进度=已完成抓包，确认网关等满 5s、inventory 无响应，下一步要查连接池和慢 SQL
```

**AI 应产出**：已完成归纳 / 下一步 1-3 条 / 验证方法。

**若没有 plan 文件**：先把昨天进度贴进对话，再说「请按排查模板帮我补一份 plan 到 Plans/Bug排查/」。

---

### Case 2b：用 MCP 先找有没有类似排查

**场景**：不确定以前是否查过类似问题。

```
知识库里有没有 API 超时、inventory 相关的排查记录？
```

命中后：

```
/resume-assistant 续做，plan=Bug排查/【AI 返回的文件名】.md，进度=...
```

---

### Case 3：写客户端技术方案

**场景**：iOS 设置页要重构，要求 Clean Arch + MVVM。

```
/template-generator 任务类型=客户端方案，背景=iOS 设置页重构，UIKit+SnapKit；痛点是 VC 过重、难测；目标可维护+单测覆盖；平台 iOS 15+
```

**存哪**：`Plans/客户端技术方案/2026-06-12-设置页重构.md`

**评审通过后**：架构决策摘要 → `Contexts/2026-06-12-设置页架构决策.md`；plan 可删。

---

### Case 4：写服务端技术方案

**场景**：订单查询 P99 飙高，考虑加缓存。

```
/template-generator 任务类型=服务端方案，背景=订单查询 P99 从 200ms 到 800ms；峰值 QPS 3k；Go 服务；需评估 Redis 缓存 vs 读库优化；写操作必须幂等
```

**存哪**：`Plans/服务端技术方案/2026-06-12-订单查询优化.md`

---

### Case 5：Code Review

**场景**：review 一个 PR，改动在聊天模块。

```
/template-generator 任务类型=review，背景=PR #1234 分支 feature/chat-model-switch，改动 NMModelComponent 与模型弹窗逻辑，关注线程安全和状态回显
```

**走查完**：阻塞项记入 plan；可复用结论 → `Contexts/2026-06-12-模型切换-review要点.md`

---

### Case 6：Figma 还原界面（链接当次贴）

**场景**：按设计稿做新弹窗，**不要把链接写进知识库**。

```
/figma-ui-assistant 新任务，Figma=https://www.figma.com/design/xxx?node-id=123-456&m=dev，平台=iOS，页面=模型选择弹窗，UIKit 项目路径 Claw/Features/ClawChat/
```

**AI 应**：读 `Contexts/Figma/` 公共规范（px÷2 等）→ 代码库搜可复用组件 → plan 存 `Plans/界面开发/`。

**做完对稿**：

```
/figma-ui-assistant 走查，plan=界面开发/2026-06-12-模型弹窗.md
```

---

### Case 7：月底复盘

**场景**：总结本月哪些任务 AI 帮上忙、哪些添乱。

```
/review-assistant 复盘时间段=2026-06-01 至 2026-06-30
```

若 plan 记得不全，补一句：

```
本月主要任务：1) 下单超时排查 用了模板+续做 2) 设置页方案 顺利 3) 某需求 AI 给了错误重试策略 已人工纠正
```

**产出**：顺利/踩坑表 + 至少 3 条改进行动 → 可存 `Contexts/2026-06-月度复盘.md`

---

### Case 9：学 LLM / 提示词（第一课，推荐起点）

**场景**：从零学提示词，**不要先学 MCP**。

```
/learn-assistant 新主题，题目=LLM基础与提示词入门，目标=能解释token和上下文窗口，并写出结构化prompt
```

**跟 plan**：`Plans/学习/2026-06-12-第1课-LLM与提示词入门.md`

**学完后**：

```
/learn-assistant 整理笔记，主题=提示词结构，要点=【你记的草稿】
```

→ 沉淀到 `Contexts/LLM学习/笔记/`

---

### Case 10：周末费曼自测（按当前进度）

```
/learn-assistant 考我，范围=本周学的 LLM基础和提示词结构，要求结合 resume_assistant 举例
```

> MCP / Agent / Skill 是后面几课；学到哪周，考哪周的范围。

---

### Case 8：会议结论随手存（不经过 Skill）

**场景**：评审会定了 HTTP 超时策略，与当前排查无关但以后会用。

Obsidian 新建：`Contexts/2026-06-15-网络超时策略会议纪要.md`，打标签 `#决策` `#会议`。

下次相关任务 AI 会通过 MCP 或 `@` 引用到。

## 5. 三层记住即可

| 层 | 文件夹 | 记什么 |
|----|--------|--------|
| 模板 | `Templates/` | 每次新任务起点 |
| 规范/结论 | `Contexts/` | 长期有效的事实与决策 |
| 进行中 | `Plans/*/` | 当前任务，**做完可删** |

**不沉淀**：Figma 链接、已上线功能清单、做完的 plan（除非你想留档）。

---

## 6. 在别的项目里用

不必打开 AI-Work-Kit。确保 `~/.cursor/skills/` 已复制分享包里的 Skill，然后在 **ClawAI / 任意仓库** 直接用：

- `/template-generator`
- `/resume-assistant`
- `/figma-ui-assistant`
- `/review-assistant`

`@Skills/xxx.md` 只有 Cursor 打开 AI-Work-Kit 文件夹时才搜得到。
